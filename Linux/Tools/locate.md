# locate

## 一、locate 是什么

`locate` 是 Linux 下用于**快速查找文件路径**的工具。它不实时扫描磁盘，而是查询**预先建立的文件名数据库**，因此速度极快。

核心特点：

* 查找速度远快于 `find`
* 适合“我记得有这个文件，但不知道在哪”
* 查的是**文件名索引**，不是实时文件系统

---

## 二、基本用法

```bash
locate 文件名
```

示例：

```bash
locate htop
```

会列出所有路径中包含 `htop` 的文件。

---

## 三、常用参数（高频）

### 1. 忽略大小写

```bash
locate -i readme
```

`README`、`readme.md` 都能匹配。

---

### 2. 只显示前 N 条结果

```bash
locate -n 10 gcc
```

避免结果过多刷屏。

---

### 3. 只匹配完整文件名

```bash
locate -b '\\bash'
```

说明：

* `-b`：只匹配路径的最后一部分
* `\\` 用于避免正则歧义

---

## 四、更新数据库（非常重要）

`locate` 查的是数据库，不是实时结果。

### 手动更新：

```bash
sudo updatedb
```

说明：

* 需要 root 权限
* 更新后才能查到新建或新拷贝的文件

很多系统通过定时任务自动更新，但开发机上**手动跑一次最可靠**。

---

## 五、locate vs find

| 对比项   | locate       | find         |
| -------- | ------------ | ------------ |
| 查找速度 | 非常快       | 慢           |
| 是否实时 | 否           | 是           |
| 查找依据 | 文件名数据库 | 实际文件系统 |
| 适用场景 | 快速定位     | 精确条件查找 |

经验法则：

> 先 `locate`，不准再 `find`

---

## 六、常见使用场景

### 1. 查找命令或工具位置

```bash
locate rsync
```

---

### 2. 查找头文件

```bash
locate stdio.h
```

---

### 3. 查找配置文件

```bash
locate sshd_config
```

---

## 七、小结

最常记住的三点：

* `locate` 很快，但不一定最新
* 查不到先 `updatedb`
* 模糊定位用它，精确筛选用 `find`

`locate` 的哲学很简单：**用空间和预处理，换取极致速度**。在需要“立刻知道在哪”的时刻，它几乎从不让人失望。
