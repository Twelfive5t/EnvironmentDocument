# htop

## 一、htop 是什么

`htop` 是一个 **交互式进程监控工具**，可以理解为 `top` 的增强版。它提供了彩色界面、鼠标操作和更直观的系统状态展示，适合日常运维和开发调试。

常见用途：

* 查看 CPU / 内存 / 负载情况
* 实时监控进程状态
* 快速结束异常进程

---

## 二、启动 htop

```bash
htop
```

如果系统未安装：

```bash
sudo apt install htop      # Debian / Ubuntu
sudo yum install htop      # CentOS / RHEL
```

---

## 三、界面说明（从上到下）

* **CPU 条**：每个 CPU 核心的使用率
* **Memory / Swap**：内存和交换分区使用情况
* **Task 信息**：进程总数、运行中进程、负载
* **进程列表**：当前所有进程

---

## 四、最常用快捷键（核心）

### 进程操作

* `↑ / ↓`：选择进程
* `F9`：结束进程（发送信号）
* `F7 / F8`：降低 / 提高进程优先级（nice）

---

### 排序与显示

* `F6`：选择排序方式（CPU、内存、时间等）
* `F5`：树形显示进程（父子关系）
* `F4`：过滤进程（按名称）

---

### 其他常用操作

* `F3`：搜索进程
* `F10`：退出 htop
* `H`：显示 / 隐藏内核线程

---

## 五、鼠标操作

* 点击 CPU / MEM 可切换排序方式
* 右键进程可弹出操作菜单
* 滚轮可滚动进程列表

---

## 六、实用使用场景

### 1. 查找占用 CPU 最高的进程

* 按 `F6`
* 选择 `PERCENT_CPU`

---

### 2. 查看进程父子关系

```text
F5  → 树形结构
```

---

### 3. 快速结束卡死进程

```text
选择进程 → F9 → SIGKILL
```

---

## 七、常用信号说明

* `SIGTERM (15)`：正常结束进程（推荐）
* `SIGKILL (9)`：强制结束进程
* `SIGHUP (1)`：重新加载配置

---

## 八、小结

日常最实用组合：

* `F6 + PERCENT_CPU`
* `F5` 查看进程树
* `F9` 结束异常进程

相比 `top`，`htop` 更直观、更顺手，是 Linux 下非常值得常驻的工具。
