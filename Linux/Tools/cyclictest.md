# cyclictest

## 一、cyclictest 是什么

`cyclictest` 是 Linux 实时性能分析工具，主要用于**测试系统调度延迟（latency）**。它通过创建周期性定时线程，测量线程被唤醒的实际时间与理论时间之间的偏差。

一句话理解：

> 用来回答“这个 Linux 系统到底有多实时？”

常见使用场景：

* 实时 Linux（PREEMPT_RT）验证
* 嵌入式系统调度延迟评估
* 驱动、内核参数调优前后对比

---

## 二、基本用法

```bash
cyclictest
```

默认行为：

* 使用 `SCHED_OTHER`
* 周期 1000 us（1ms）
* 输出最小 / 平均 / 最大延迟

---

## 三、最常用参数（核心）

### 1. 设置线程数量

```bash
cyclictest -t4
```

创建 4 个测试线程，常用于多核系统。

---

### 2. 设置测试周期（us）

```bash
cyclictest -p 99 -i 1000
```

说明：

* `-i`：周期，单位微秒
* `-p`：实时优先级（1–99）

这是测试实时系统的典型组合。

---

### 3. 绑定 CPU（避免调度干扰）

```bash
cyclictest -a 0
```

只在 CPU0 上运行。

---

### 4. 使用 FIFO 调度策略

```bash
cyclictest -m -S -p 80
```

参数说明：

* `-S`：使用 `SCHED_FIFO`
* `-m`：锁定内存，避免 page fault

---

## 四、常见推荐组合

### 实时系统基准测试

```bash
sudo cyclictest -p 99 - m -c 0 -i 1000 -n -h 100 -q -l 1000000 -t 4 -a 0-3
```

含义：

* -p 99 为给cyclictest 赋优先级99
* -m 为锁定内存分配，
* -c 0 为指定使用默认的MONOTONIC 时钟，
* -i 1000 为指定一个循环为1000us，
* -l 1000000 为总共1000000个循环，
* -n 为使用nanosleep 而不是简单的sleep，
* -q 为在运行时不打印即时信息
* -h 100 为总共统计100个信息在最后的结果中。
* -t 4 为创建4个实时线程
* -a 0-3 为可用 CPU 列表是 CPU0、1、2、3

---

## 五、输出结果解读

示例输出：

```text
T: 0 ( 1234) P:99 I:1000 C:100000 Min:2 Act:5 Avg:7 Max:45
```

关键字段说明：

* `Min`：最小延迟（us）
* `Avg`：平均延迟（us）
* `Max`：最大延迟（us）

评估实时性的核心指标通常是 **Max latency**。

---

## 六、典型延迟参考（经验值）

* 普通 Linux：几十到几百 us
* PREEMPT_RT：十几 us 以内
* 高负载下仍低于 50 us：实时性较好

具体标准需结合业务需求判断。

---

## 七、注意事项

* 建议使用 root 权限运行
* 测试时关闭无关后台任务
* 对比测试需保持参数一致

---

## 八、小结

`cyclictest` 的关注点只有一个：

> **最坏情况下，系统会慢到什么程度？**

在实时系统里，平均值不重要，最大值才是真相。
